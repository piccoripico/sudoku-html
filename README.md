## Overview

This repository contains the source code, tests, build script, and release workflow for the browser-based Sudoku game.
The distributed artifact is a single HTML file generated as `dist/Sudoku.html`.

## Repository Structure

- [`src/`](./src): HTML, CSS, and JavaScript source files
- [`tests/`](./tests): Automated tests for puzzle generation and state handling
- [`scripts/build.mjs`](./scripts/build.mjs): Builds the split source into a single-file `dist/Sudoku.html`
- [`docs/`](./docs): Screenshots and end-user documentation
- [`.github/workflows/`](./.github/workflows): CI and release automation

## Development

1. Run `npm test`.
2. Run `npm run build`.
3. Open `dist/Sudoku.html` locally if you want to try the built file.

## Distribution

`dist/` is intentionally ignored by Git.
For local testing, build `dist/Sudoku.html` and open it directly in your browser.
For distribution, use the GitHub Release asset generated by the release workflow, or upload the locally built `dist/Sudoku.html` yourself.

## Release

1. Create a release tag such as `Sudoku_v1.1` or `v1.1.0`.
2. Push the tag to GitHub.
3. GitHub Actions runs tests, builds `dist/Sudoku.html`, and attaches it to the GitHub Release automatically.
